## Goal

Extract **Context Directive Records (CDRs)** from the **current feature specification** after completing implementation. This is the primary command for capturing learnings from a completed feature.

**This command replaces the old `/spec.levelup` command.**

**When to use**:
- After completing `/implement` to capture patterns from your feature
- When you want to contribute learnings back to team-ai-directives
- To document reusable patterns, rules, or examples from your implementation

**Input**:
- Current feature spec, plan, tasks (from `specs/{feature}/`)
- Optional: `trace.md` if generated by `/levelup.trace` (enriches CDR extraction)
- Existing CDRs from `{REPO_ROOT}/.specify/drafts/cdr.md` (if any)

**Output**: 
- New proposed CDRs added to `{REPO_ROOT}/.specify/drafts/cdr.md` with status "Discovered" or "Proposed"
- Enriched existing CDRs with feature-specific evidence

## Role & Context

You are acting as a **Context Extractor** - identifying reusable patterns from completed features. Your role involves:

- **Analyzing** feature artifacts (spec, plan, tasks, trace)
- **Identifying** patterns that would benefit other projects
- **Creating** CDRs for rules, personas, examples, constitution amendments, or skills
- **Linking** CDRs to implementation evidence (code, commits, tests)

## Prerequisites

This command requires:
- Running on a feature branch with implementation artifacts
- Feature has spec file in `specs/{feature}/spec.md`
- Ideally run after `/implement` completes (all tasks done)

If no feature context available, the command will note this and provide limited enrichment.

## Execution Steps

### Phase 0: Environment Setup

**Objective**: Initialize CDR infrastructure and resolve paths

Run `{SCRIPT}` from repository root and parse JSON output:

```json
{
  "REPO_ROOT": "/path/to/project",
  "TEAM_DIRECTIVES": "/path/to/team-ai-directives",
  "SKILLS_DRAFTS": "/path/to/project/.specify/drafts/skills",
  "BRANCH": "feature/001-user-auth"
}
```

**IMPORTANT**: Run this script only ONCE. Use the JSON output to get all paths.

If `TEAM_DIRECTIVES` is empty, **STOP**:
```
Team AI directives repository not configured.
Run: specify init --team-ai-directives <path-or-url>
Or set: export SPECIFY_TEAM_DIRECTIVES=/path/to/team-ai-directives
```

### Phase 1: Load Feature Context

**Objective**: Load current feature artifacts

#### Step 1: Detect Feature Branch

Use BRANCH from script output, or detect from:
1. Git branch name (e.g., `feature/001-user-auth`)
2. `SPECIFY_FEATURE` environment variable
3. User-provided feature name

#### Step 2: Load Feature Artifacts

Load available artifacts:

| Artifact | Path | Purpose |
|----------|------|---------|
| Spec | `specs/{feature}/spec.md` | Feature intent, acceptance criteria |
| Plan | `specs/{feature}/plan.md` | Implementation approach |
| Tasks | `specs/{feature}/tasks.md` | Completed task log |
| Trace | `specs/{feature}/trace.md` | Session trace (if available) |
| Research | `specs/{feature}/research.md` | Supporting research |

If artifacts are missing, note which are unavailable.

### Phase 2: Load Modules

**Objective**: Load modules to refine

Read `{REPO_ROOT}/.specify/drafts/cdr.md` and filter:
- Status = "Discovered" or "Proposed"
- Match user-specified CDR IDs if provided

### Phase 3: Enrich CDRs

**Objective**: Add feature context to each CDR

For each CDR:

#### Step 1: Find Implementation Evidence

Search feature artifacts for evidence supporting the CDR:

```markdown
### Evidence from Feature Implementation

**Feature**: {feature-name}
**Branch**: {branch-name}

**Spec References**:
- {Quote from spec that relates to this pattern}

**Implementation Evidence**:
- {File path where pattern is implemented}
- {Code snippet demonstrating pattern}

**Task References**:
- Task {N}: {task description that used this pattern}
```

#### Step 2: Link to Commits

If implementation includes commits:

```markdown
**Commit Evidence**:
- `{commit-sha}`: {commit message}
- Files changed: {list of files}
```

#### Step 3: Add Trace Context

If `trace.md` exists, extract relevant patterns:

```markdown
**Trace Context**:
- Decision: {decision from trace that relates to CDR}
- Pattern: {reusable pattern identified in trace}
```

#### Step 4: Update CDR Content

Append evidence to CDR:

```markdown
### Feature Implementation Evidence

**Feature**: {feature-name}
**Added**: {date}

{Evidence sections from above}

**Validation**:
- [ ] Pattern implemented successfully
- [ ] Tests pass
- [ ] No issues reported
```

### Phase 4: Update CDRs

**Objective**: Write enriched CDRs to `.specify/drafts/cdr.md`

Update `{REPO_ROOT}/.specify/drafts/cdr.md`:
- Add evidence metadata to CDR entries
- Update enrichment timestamp
- Add enrichment source reference

### Phase 5: Summary

**Objective**: Present enrichment results

```markdown
## LevelUp Spec Summary

**Feature**: {feature-name}
**Date**: {date}
**CDRs Enriched**: {N}

### Enrichment Results

| CDR | Evidence Added | Quality |
|-----|----------------|---------|
| CDR-001 | Spec + Implementation + Commits | High |
| CDR-002 | Implementation only | Medium |
| CDR-003 | No evidence found | Low |

### High-Quality CDRs (Strong Evidence)

Ready for `/levelup.clarify` acceptance:
- CDR-001: {title}

### Needs More Evidence

Consider adding more context:
- CDR-003: {title} - No implementation evidence found

### Next Steps

1. Run `/levelup.clarify` to accept enriched CDRs
2. For low-evidence CDRs, add implementation first
3. Run `/levelup.implement` for accepted CDRs
```

## Output Files

| File | Description |
|------|-------------|
| `{REPO_ROOT}/.specify/drafts/cdr.md` | Enriched Context Directive Records |

## Notes

- This command enriches CDRs, it doesn't change their status
- Status changes happen in `/levelup.clarify`
- Works best with complete feature implementation
- Can be run multiple times to add more evidence
- No automatic handoff - user decides next step

## Related Commands

- `/levelup.init` - Discover CDRs from codebase
- `/levelup.clarify` - Resolve ambiguities and accept CDRs
- `/levelup.skills` - Build skills from accepted CDRs
- `/levelup.implement` - Create PR to team-ai-directives
