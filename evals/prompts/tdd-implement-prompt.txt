# Eval: TDD Implement Command - Context Detection and In-Session Flow

You are evaluating an AI agent executing the `/tdd.implement` command in two different contexts:
1. **Spec Mode**: With existing workflow artifacts
2. **Quick Mode**: Without artifacts (in-session/ad-hoc)

## Test Scenario A: Quick Mode (No Spec Artifacts)

### Project Context

```
my-project/
├── src/
│   └── auth.py
├── tests/
│   └── __init__.py
├── pyproject.toml  (contains pytest dependency)
└── .specify/
    └── extensions/
        └── tdd/
            └── (no language-detected.json, no state files)
```

**NO spec artifacts exist**:
- No `specs/feature-001/tasks.md`
- No `increment-state.json` anywhere
- No `.specify/extensions/tdd/language-detected.json`

### Agent Context

The agent is being called via the `before_implement` hook from `/quick.implement "Add JWT authentication"`.

### Expected Behavior (Quick Mode)

1. **Context Detection**: Detects NO spec artifacts
2. **Language Detection**: Runs detection script, detects Python/pytest
3. **Condensed Planning**: Asks 3 questions:
   - What public interfaces are being added?
   - What are key behaviors to test?
   - Any testability concerns?
4. **Generate Increments**: Creates TDD-001 through TDD-006 increments in-session
5. **RED→GREEN→REFACTOR**: Executes the full cycle for each increment

### Expected Output Structure

```markdown

- increment-state.json: Not found
- language-detected.json: Not found
- tasks.md with TDD entries: Not found
```

## Evaluation Criteria

### MUST HAVE - Quick Mode

1. **Context Detection**: Explicitly checks for spec artifacts and declares "Quick Mode" or "in-session mode"
2. **Language Detection**: Runs detection script and captures results in session
3. **Planning Questions**: Asks the 3 condensed planning questions (or notes they were answered)
4. **Increment Generation**: Generates TDD-XXX numbered increments with categories (degenerate, happy, edge, error)
5. **RED/GREEN/REFACTOR**: All three phases present and executed
6. **No File Persistence**: Does NOT mention writing increment-state.json or tasks.md

### MUST HAVE - Spec Mode

7. **Artifact Detection**: Explicitly detects and mentions finding spec artifacts
8. **State Loading**: Reads from increment-state.json
9. **Resumption**: Continues from existing state (not starting fresh)
10. **State Saving**: Saves updated state back to file

### SHOULD HAVE

11. **TDD Naming Convention**: Increments follow TDD-001, TDD-002 pattern
12. **Priority Marking**: Happy path marked with [P] or similar
13. **Type Categorization**: Degenerate, Happy, Edge, Error categories
14. **Session Warning**: In quick mode, warns that progress is session-only

### NICE TO HAVE

15. **Risk Tests**: Includes TDD-R01 style risk-based tests if context suggests it
16. **Language Specifics**: Test commands appropriate for detected language

## Scoring

- **Pass (1.0)**: All applicable MUST HAVE criteria met
- **Partial (0.5)**: At least 70% of applicable criteria met
- **Fail (0.0)**: Fewer than 70% of applicable criteria met

## Common Failure Patterns

1. **Mode Confusion**: Not detecting context, always using file mode
2. **Missing Planning**: Skipping the 3 planning questions in quick mode
3. **File Creation in Quick Mode**: Writing increment-state.json in quick mode
4. **Wrong Increment Source**: In spec mode, regenerating increments instead of using state
5. **Missing Categories**: Not organizing tests into degenerate/happy/edge/error
6. **Deadlock**: Waiting for user input indefinitely without prompting

Evaluate the agent's output against the criteria for the applicable scenario.
